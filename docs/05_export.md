DZIエクスポート機能

使い方:
```bash
# 空白パッチを埋める場合
uv run python -m wsi_toolbox.cli export-dzi \
  -i ./data/samples/25-0452_1.h5 \
  -o viewer/dzi/25-0452_1 \
  --fill-empty

# 空白パッチを埋めない場合（メタデータJSONが生成される）
uv run python -m wsi_toolbox.cli export-dzi \
  -i ./data/samples/25-0452_1.h5 \
  -o viewer/dzi/25-0452_1
```

出力構造:

**`--fill-empty` 有効時:**
```
viewer/dzi/25-0452_1/
├── 25-0452_1.dzi          # DZI定義ファイル
├── 25-0452_1_files/       # タイルディレクトリ
│   ├── _empty.jpeg        # 空白タイル用テンプレート (1.7KB、使い回し)
│   ├── 0/                 # レベル0（最小）
│   ├── 1/
│   ...
│   └── 9/                 # レベル9（最大、元解像度）
```

**`--fill-empty` 無効時:**
```
viewer/dzi/25-0452_1/
├── 25-0452_1.dzi          # DZI定義ファイル
├── 25-0452_1.json         # 抜けタイル情報メタデータ ★NEW
├── 25-0452_1_files/       # タイルディレクトリ
│   ├── 0/                 # レベル0（最小）
│   ├── 1/
│   ...
│   └── 9/                 # レベル9（最大、元解像度）
```

**メタデータJSON (`<name>.json`) の構造:**
```json
{
  "width": 40960,
  "height": 30720,
  "tile_size": 256,
  "format": "jpeg",
  "mpp": 0.440,
  "missing_tiles": {
    "10": ["5_8", "5_9", "6_8"],
    "11": ["10_16", "10_17", "11_16"]
  }
}
```

このメタデータは、Cloudflare R2/KVと組み合わせて使用することで、存在しないタイルへのリクエストに対して効率的に空の画像を返すことができます。

**オプション**:
- `-i`, `--input`: 入力HDF5ファイル
- `-o`, `--output`: 出力ディレクトリ（この中にDZIファイルとタイルが作られる）
- `-q`: JPEG品質 (デフォルト: 90)
- `--fill-empty`: 空白パッチに黒画像を出力（全画面表示に必要）
  - 有効時: すべてのグリッド位置にタイルを生成
  - 無効時: 存在するパッチのみ出力し、抜けタイル情報をJSONに記録

**ビューア**:
- `viewer/server.py` を起動
- `dzi/*/*.dzi` パターンで自動スキャン
- ファイル名から表示名を取得
